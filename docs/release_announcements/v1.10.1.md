# flux v1.10.1 is now live

This is a coordinated **patch release** tightening the schema-system features
introduced in `v1.10.0`. Only `fluxCore`, `fluxForecast`, and the super-repo
move to `v1.10.1`; the other five subrepos remain at `v1.10.0` (their
dependency floors stay `(>= 1.10.0)`).

## What changed

### fluxCore (1.10.1)

- **Removed the `id_string` type** outright (no deprecation alias). Use
  `nonempty_string` or a custom `validate` function for identifier columns.
  The supported type list is now 14 entries (correction to v1.10.0's
  "14 types" headline post `id_string` removal + `percent` addition).
- **Added the `percent` type**: numeric in `[0, 100]`, honors optional
  `min` / `max` overrides — analog of `probability` but on the 0–100 scale.
- **Rewrote `set_schema()`** with hybrid `vars` syntax. Each element of
  `vars` can now be either a type-name string (e.g. `"count"`) or a fully
  specified list (e.g. `list(type = "positive_numeric", max = 20)`); both
  shapes can be mixed in one call. The previous `replace = ` / `add = `
  flags were replaced by a cleaner `overwrite = FALSE` (errors on collision
  unless `TRUE`) plus a `remove = ` argument that drops named entries
  (errors if absent).
- **Explicit permissive-by-default policy**: built-in type validators
  enforce only what the type semantic implies (e.g. `count` ≥ 0,
  `probability` ∈ [0,1]); use `min` / `max` or a custom `validate` to
  tighten further. Documented in `subrepos/fluxCore/docs/schema_spec.md`.

### fluxForecast (1.10.1)

- `validate_forecast()` now **delegates schema validation to
  `fluxCore::schema_validate()`** — fluxCore is the single source of truth
  for what a valid schema looks like. The previously inline `allowed_types`
  list and per-var `$levels` re-check have been removed. Future fluxCore
  type additions/removals propagate automatically.
- Dependency floor bumped to `fluxCore (>= 1.10.1)`.

### Super-repo (1.10.1)

- Reorganized: `header_logo.png` moved to `resources/images/`; the release
  and maintenance scripts moved to `resources/scripts/release/` and
  `resources/scripts/maintenance/`. Vestigial top-level `reports/` directory
  removed. README, Makefile, and `.Rbuildignore` rewired accordingly.
- Tutorial `tutorials/01_core_engine_scaffold.Rmd` refreshed: the
  `set_schema()` example now demonstrates the hybrid syntax (categorical
  via `levels` list, `percent`, `positive_numeric` with `max`, `count`,
  `probability`) and uses the result variable name `quicker_schema`.
