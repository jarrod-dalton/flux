# flux v1.10.0 is now live

We've completed the `v1.10.0` ecosystem release for **flux**.

`flux` is a modular R ecosystem for simulation modeling of probabilistic dynamic systems in irregular time, with coordinated packages for:

- core simulation engine
- train/test/validation (TTV) data preparation
- forecasting and summary estimators
- prediction-vs-observed validation
- multi-process orchestration
- reference model and template scaffolding

## Highlights in v1.10.0

- **Expanded schema type system**: 14 built-in variable types (`logical`,
  `binary`, `integer`, `count`, `nonnegative_integer`, `positive_integer`,
  `numeric`, `nonnegative_numeric`, `positive_numeric`, `probability`,
  `categorical`, `ordinal`, `string`, `nonempty_string`, `id_string`) with
  built-in validation strategies.
- **Automatic type-specific defaults**: `default` and `coerce` are now
  optional in schema specifications; fluxCore applies appropriate defaults
  based on declared type (e.g. `as.numeric` for numeric types, `NA_real_`
  for numeric defaults).
- **New `set_schema()` helper** in fluxCore: simplified schema creation via
  a named character vector of type mappings; reduces boilerplate for common
  cases.
- **Coordinated downstream alignment**: fluxPrepare, fluxForecast,
  fluxValidation, fluxOrchestrate, fluxModelTemplate, and fluxASCVD updated
  to consume the new type system and bumped to `1.10.0`.
- **Synchronized dependency floors**: all flux-internal `Imports` and
  `Suggests` floors raised to `(>= 1.10.0)`.
- **Super-repo polish**: header logo, refreshed release-line messaging,
  and updated release tooling defaults.
